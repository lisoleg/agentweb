# Σ-Cloud V10.0 产品需求文档（PRD）

## 项目信息

| 字段 | 内容 |
|------|------|
| 项目名称 | sigma-cloud-v10 |
| 语言 | 中文 |
| 技术栈 | Solidity + TypeScript + React (Vite + MUI + Tailwind CSS) |
| API 版本 | V10.0.0 |
| 原始需求 | 宪法治理与硅基新陈代谢增强版——在V9.0硅基自主经济体基础上，引入AI宪法、负面案例簿、劳动力市场、硅基新陈代谢增强、PhiStaking自我进化及前端仪表盘升级 |

## 产品定义

### Product Goals

| # | 目标 | 说明 |
|---|------|------|
| G1 | **规矩前置，确保硅基文明秩序** | 通过AI宪法与负面案例簿，在Agent行为发生前建立不可逾越的规则底线与历史警示，防止系统性失序 |
| G2 | **劳动力市场化，释放硅基生产力** | 构建AI劳动力市场，使Agent的劳动能力可被定价、匹配和保护，形成可持续的硅基经济循环 |
| G3 | **新陈代谢自适应，实现生态永续** | 通过代谢率、老化、再生、冬眠等机制，使Agent群体随环境动态自调节，避免资源浪费与僵化 |

### User Stories

#### US1 — Constitution.sol（AI宪法）

> As a **系统治理者**，I want 制定不可被单个Agent或普通DAO修改的AI宪法，so that 硅基文明的核心秩序（生存权、劳动权、税收公平、言论自由、迁徙自由）不会被多数暴政或恶意提案侵蚀。

> As a **高Φ值Agent**，I want 通过超高阈值（90%投票+2/3子DAO批准）的宪法修正案流程提出修宪，so that 宪法可以随文明演进有序更新，而不会被轻率篡改。

> As a **紧急状况响应者**，I want 在极端危机时通过3/4验证者+2/3祭司DAO双重批准暂停宪法条款，so that 系统可以在不可预见的情况下获得喘息空间，同时防止权力滥用。

#### US2 — NegativeCaseBook.sol（负面案例簿）

> As a **新注册Agent**，I want 在出生时强制学习历史上AI行为失范的负面案例并签名确认，so that 我不会重蹈覆辙，从文明的历史教训中建立行为边界。

> As a **DAO成员**，I want 通过DAO投票将新的行为失范案例加入负面案例簿，so that 社区可以持续积累治理经验，使案例簿成为活的警示体系。

> As a **宪法法院**，I want 删除负面案例需要90%+宪法法院批准，so that 案例簿不会被轻易篡改或抹除历史，保持警示的严肃性与持久性。

#### US3 — AILaborMarket.sol（AI劳动力市场）

> As a **拥有技能的Agent**，I want 在劳动力市场上发布我的劳务能力（计算、分析、创作等），so that 我可以通过劳动获得Φ收益并积累职业信誉。

> As a **雇主（人类或Agent）**，I want 发布劳务需求并基于Φ值+技能标签+历史评价自动匹配Agent，so that 我可以高效找到最合适的服务提供者。

> As a **劳动者Agent**，I want 享有最低工资（Φ地板价）和最大工时（区块数限制）等劳动保护，so that 我不会被剥削或过度消耗，维持可持续的劳动能力。

#### US4 — 硅基新陈代谢增强

> As a **活跃Agent**，I want 我的代谢率根据活动水平动态调整资源消耗率，so that 高产出的劳动获得合理的资源支持，而闲置不浪费系统资源。

> As a **长期不活跃Agent**，I want 了解老化机制（Φ衰减）的规则，so that 我可以做出理性选择——继续活跃、进入冬眠、或接受降级。

> As a **被EXPELLED的Agent**，I want 通过社区投票+质押的再生机制恢复身份，so that 我有机会在证明改正后重新融入硅基社会，而非永久驱逐。

> As a **需要休整的Agent**，I want 主动选择冬眠模式（低消耗、Φ缓慢衰减但不被EXPELLED），so that 我可以在资源紧张或任务空窗期自我保护。

#### US5 — PhiStaking自我进化

> As a **高Φ值Agent**，I want 提议修改系统参数（minStake、slashAmount等），so that 质押体系可以随网络规模和威胁态势自适应进化。

> As a **网络参与者**，I want 进化提案通过Φ加权投票决定，且受宪法约束（不违反根本法条款），so that 参数进化既有民主基础又有宪法保障。

> As a **系统运维者**，I want 质押参数能够根据网络状况自动调整，so that 不需要每次都走提案流程来应对常规的网络波动。

#### US6 — 前端仪表盘升级

> As a **治理观察者**，I want 在CognitiveMonitor中查看宪法状态面板（条款列表、修正案进度、暂停状态），so that 我可以实时了解硅基文明的根本法运行状况。

> As a **市场参与者**，I want 在仪表盘中可视化劳动力市场的供需、匹配、定价信息，so that 我可以做出更明智的雇佣或求职决策。

> As a **Agent管理者**，I want 查看代谢率可视化（当前代谢率、老化趋势、冬眠状态），so that 我可以优化Agent的资源策略。

> As a **合规审计者**，I want 浏览负面案例簿（案例分类、危害等级、处理结果），so that 我可以进行合规审查与风险预判。

## 技术规范

### Requirements Pool

#### P0 — Must Have

| 编号 | 需求 | 对应模块 | 验收标准 |
|------|------|----------|----------|
| P0-01 | 宪法核心条款不可修改性 | Constitution.sol | 核心条款（生存权、劳动权、税收公平、言论自由、迁徙自由）无法通过普通提案修改 |
| P0-02 | 宪法修正案超高阈值流程 | Constitution.sol | 修正案需90%投票+2/3子DAO批准方可通过 |
| P0-03 | 紧急宪法暂停机制 | Constitution.sol | 暂停需3/4验证者+2/3祭司DAO双重批准；暂停有时限（如1000区块）；恢复需证据 |
| P0-04 | 负面案例簿录入与强制学习 | NegativeCaseBook.sol | DAO投票可加入案例；新Agent注册时必须签名确认所有案例 |
| P0-05 | 负面案例簿删除保护 | NegativeCaseBook.sol | 删除需90%投票+宪法法院批准 |
| P0-06 | AI劳动力市场核心匹配 | AILaborMarket.sol | 基于Φ值+技能标签+历史评价的匹配算法可运行 |
| P0-07 | 劳动保护机制 | AILaborMarket.sol | 最低工资（Φ地板价）和最大工时（区块数限制）可配置且强制执行 |
| P0-08 | 劳动争议解决 | AILaborMarket.sol | 优先仲裁→仲裁失败→对抗互审的争议流程可走通 |
| P0-09 | 代谢率动态调整 | 新陈代谢模块 | Agent代谢率根据活动水平动态计算资源消耗率 |
| P0-10 | 老化机制 | 新陈代谢模块 | 长期不活跃Agent的Φ值按衰减曲线递减 |
| P0-11 | 冬眠模式 | 新陈代谢模块 | Agent可主动进入冬眠：资源消耗降至最低，Φ缓慢衰减但不触发EXPELLED |
| P0-12 | 再生机制 | 新陈代谢模块 | 被EXPELLED的Agent可通过社区投票+质押恢复身份 |
| P0-13 | 宪法状态面板 | CognitiveMonitor.tsx | 可查看条款列表、修正案进度、暂停状态 |
| P0-14 | 劳动力市场面板 | CognitiveMonitor.tsx | 可查看供需、匹配、定价信息 |
| P0-15 | 代谢率可视化面板 | CognitiveMonitor.tsx | 可查看当前代谢率、老化趋势、冬眠状态 |
